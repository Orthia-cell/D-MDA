🔢 The Calculus Connection

At its core, this project is about integration — one of the two fundamental operations in calculus.

# The ODE Being Solved:
dy/dt = sin(t) → The rate of change follows a sine wave

# The Analytical Solution:
y = 1 - cos(t) → The accumulated value over time

# Physical Realization:
• Disk rotation represents time (t)
• Wheel position represents sin(t)
• Wheel rotation accumulates the integral (1 - cos(t))

The wheel doesn't "calculate" the integral — it is the integral. This is the essence of analog computing: physical systems embody mathematical relationships.

📈 Project Phases

Phase 1: Physics Environment

Established the basic differentiable physics simulation using Taichi. Set up disk, wheel, and shaft bodies with proper inertia and contact detection.

  • Differentiable simulation framework
  • Rigid body dynamics
  • Contact detection

Phase 1: Basic physics simulation showing disk rotation and wheel dynamics

Phase 2: Rolling Constraint

Implemented the rolling-without-slipping constraint using velocity damping. Fixed Taichi compiler issues and achieved 0.0003m position constraint accuracy.

  • No-slip condition implementation
  • Velocity damping approach
  • Position constraint stabilization

Phase 2: Optimization convergence showing loss landscape exploration

Phase 3: Parameter Optimization

Used gradient descent to find the optimal wheel offset. Discovered that 0.400m produces the best integration accuracy.

  • Grid search over offset range
  • Finite difference gradients
  • Loss: 0.0096, Error: 0.098

Phase 3: Loss landscape and ODE integration at optimal 0.400m offset

Phase 4: Architecture Search (Evolutionary)

The breakthrough: genetic algorithm discovered a 2-wheel series configuration that outperforms any single-wheel design. Evolution found what humans wouldn't intuit.

  • Population: 15 mechanisms
  • Generations: 10
  • Discovered: 2 wheels, series connection
  • Loss: 0.000024 (400× better!)

Phase 4: Evolutionary discovery showing 400× performance improvement

💡 Key Insights

AI Can Design Physical Systems: We didn't design the winning 2-wheel mechanism — an evolutionary algorithm did. It explored a design space, evaluated candidates, and converged on a non-obvious solution that outperforms human designs.

The Power of Evolutionary Search

The genetic algorithm discovered that a 2-wheel series configuration with specific offsets and gear ratios outperforms any single-wheel design. This topology would not have been found through manual tuning!

Analog Computing Renaissance

In an age of digital dominance, D-MDA suggests that analog computing — evolved, optimized, and domain-specific — may be due for a renaissance. The energy efficiency of analog (continuous physical computation vs. billions of digital switching operations) offers a path to sustainable computing for suitable problems.

🔬 Technical Contributions

Differentiable Physics

Implemented a physics engine where every parameter is differentiable — enabling gradient-based optimization through the physical simulation. This bridges the gap between "what the math says" and "what the machine does."

Evolutionary Architecture Search

Rather than tuning a fixed design, we searched the space of possible designs:

  • Number of components (1-3 wheels)
  • Connection topology (series, parallel, compound)
  • Parameter values (offsets, gear ratios)

Quantified Performance Gains

The evolutionary approach achieved a 400× improvement in loss and 20× improvement in error over hand-tuned designs.
